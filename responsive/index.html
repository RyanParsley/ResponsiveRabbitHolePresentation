Responsive Web Design

“Fluid grids, flexible images, and media queries are the three technical ingredients for responsive web design, but it also requires a different way of thinking.”

― Ethan Marcotte

TLDR

  1. Responsive is a loaded term

    Think about the results more than the approach

  2. Don’t get lazy

    Always question if this approach is really serving the needs of your users

  3. …but don’t get over zealous

    Don’t get more clever than you can properly test. When in doubt, err on the side of simplicity.

You used to be much more… “muchier.” You’ve lost your muchness.

  • Don’t pretend to know what your users want

  • There are many phones of many sizes

  • Soon there will be more stuff with more screen sizes

  • Don’t be a hero!

If you don’t know where you are going, any road will get you there.

  • Any approach is just as viable as any other provided you don’t know anything about your users.

  • If you know better (and I really hope you do) do something else.

How to make an inch an inch

  • Menu > Image > Image Size

    Make sure “Resample Image” is not active

  • This looks like a job for… Math!

    Remember our old friend Target/Context = Result?

  • [Monitor dpi]/[Document dpi] = Zoom

    Example: Most monitors have a resolution of 96dpi. To make a document with the same resolution as an iPhone 4 (326) appear on screen as the right physical dimensions you would need to zoom at 29.63%. (96/324=29.63)

Work as big as you anticipate needing

  • Images shrink better than they scale

  • Sometimes they don’t shrink so well

Typography gotchas

  • You may want to change border widths to reflect font size changes

  • Be careful with your line heights

  • Watch out for awkwardly narrow columns

  • With all these moving parts consistency is more important than ever

  • Navigation links don’t all need to be full width buttons just because you’re using media queries

  • Some of those widows can be prevented

Image gotchas

  • Don’t let them get too big.

    Remember that “too big” means half size in Retna Duplo mode.
